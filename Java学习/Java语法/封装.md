# 封装

## 介绍

封装是把抽象出的数据属性和对数据的操作方法封装在一起，数据被保护在内部，程序的其他部分只能通过被授权的方法来对数据进行操作。

## 好处

- 隐藏实现细节
- 可以对数据进行验证，保证安全合理

## 实现步骤

1、将属性进行私有化private

2、提供一个公共的public set方法，用于对属性判断并赋值

```
public void setXxx(类型 参数名){//Xxx表示某个属性

​	//加入数据验证的业务逻辑

​    属性 = 参数名；

}
```

3、提供一个公共的public get方法，用于获取属性的值

```
public 数据类型 getXxx(){//权限判断

​    return xx;

}
```

### 示例

```
public class Person {
    public String name;
    private int age;
    private double salary;

    public String getName() {
        return name;
    }
    

    public void setName(String name) {
        if (name.length() >= 2 && name.length() <= 6){
            this.name = name;
        }
        else {
            System.out.println("name is error!");

        }

    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        if (age < 1 || age > 100){
            System.out.println("age is error!");
            this.age = 18;
        }else{
            this.age = age;
        }

    }

    public double getSalary() {
        return salary;
    }

    public void setSalary(double salary) {
        this.salary = salary;
    }
}
在set方法中来验证age和name的范围等。
```

有一个问题：当Person类中使用了带参数的构造函数后，可以直接绕过set方法的验证。

## 封装与构造器

```
public Person() {
}

public Person(String name, int age, double salary) {
    this.name = name;
    this.age = age;
    this.salary = salary;
}
```

当使用这样的带参构造函数后，可以直接获取到name、age、salary的值，就没有了校验的功能。

因此，可以将set方法放到构造函数中，将构造方法和set函数结合，让它们在赋值之前通过校验，代码如下

```
public Person(String name, int age, double salary) {
        setSalary(salary);
        setName(name);
        setAge(age);
    }
```
